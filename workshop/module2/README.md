# Module 2: Refactor & Containerize
In this module, you will be learning to containerize the `asset-manager` app using the project documentation & specifications available to you and generating Dockerfiles optimized for Java workloads with GitHub Copilot.

## Project documentation
**Web module** (`web/`) is a Spring Boot MVC application with the following components:
  - `S3Controller.java` — 6 Thymeleaf endpoints (list, upload form, upload handler, view page, download, delete)
  - `StorageService.java` interface — abstraction over AWS S3 (`AwsS3Service`) and local filesystem (`LocalFileStorageService`)
  - 4 Thymeleaf templates — `list.html` (image gallery with polling), `upload.html` (drag-drop form), `view.html` (image viewer), `layout.html` (master layout)
  - JPA layer — `ImageMetadata` entity, `ImageMetadataRepository`
  - RabbitMQ integration — sends `ImageProcessingMessage` on upload

**Worker module** (`worker/`) is a background processing service that
  - Listens to `image-processing` RabbitMQ queue
  - Downloads the original image and generates thumbnail (max 600px, preserving aspect ratio)
  - Uploads thumbnail back to storage, updates metadata in database
  - Uses manual acknowledgment (`channel.basicAck` / `channel.basicNack`)

**Current endpoints (all return HTML):**

| Method | Endpoint | Returns | Purpose |
|--------|----------|---------|---------|
| GET | `/` | Redirect | Redirects to `/storage` |
| GET | `/storage` | `list.html` | Image gallery grid |
| GET | `/storage/upload` | `upload.html` | Upload form |
| POST | `/storage/upload` | Redirect | Process upload, send to queue |
| GET | `/storage/view-page/{key}` | `view.html` | Single image viewer |
| GET | `/storage/view/{key}` | Binary stream | Download image bytes |
| POST | `/storage/delete/{key}` | Redirect | Delete image + metadata |


## Refactor the API layer

### Design new REST API contracts
- Use the GitHub Copilot Chat, refactor the Thymeleaf-based controller into a JSON REST API.
- Map each Thymeleaf endpoint to a REST equivalent.
- Verify the API contract generated by AI and make corrections if needed.


### Create a new `ImageApiController`
1. Navigate to `web/src/main/java/.../controller/` and create a new file titled `ImageApiController.java`
- Annotate with `@RestController` and `@RequestMapping("/api/images")`
- Inject `StorageService` (same interface the Thymeleaf controller uses)
- Implement each REST endpoint returning `ResponseEntity<>`

2. Create `ImageResponse.java` DTO in `web/src/main/java/.../dto/`:
- Map fields from `S3StorageItem` and `ImageMetadata`

3. **Add CORS configuration** in `WebMvcConfig.java`:
   ```java
   @Override
   public void addCorsMappings(CorsRegistry registry) {
       registry.addMapping("/api/**")
               .allowedOrigins("http://localhost:4200")
               .allowedMethods("GET", "POST", "DELETE", "OPTIONS");
   }
   ```

4. **Key implementation details:**
   - `GET /api/images` — calls `storageService.listObjects()`, maps to `List<ImageResponse>`
   - `POST /api/images` — accepts `MultipartFile`, calls `storageService.uploadFile()`, returns created resource
   - `DELETE /api/images/{key}` — calls `storageService.deleteObject()` for original + thumbnail, deletes metadata


5. Test and verify all API endpoints are working as expected.
- Acceptance criteria: confirm the `worker` still generates thumbnails (check RabbitMQ management at http://localhost:15672).
